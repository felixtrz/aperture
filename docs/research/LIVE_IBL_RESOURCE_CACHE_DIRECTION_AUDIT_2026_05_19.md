## Findings

The WebGPU app already has renderer-owned caches for:

- source texture GPU resources: `WebGpuAppResourceCache.textures`;
- source sampler GPU resources: `WebGpuAppResourceCache.samplers`;
- prepared mesh GPU buffers;
- prepared material GPU resources;
- per-family frame resources; and
- the canvas depth attachment.

The material texture/sampler cache keys are currently source-asset/version
based through `sourceAssetCacheKey()`. That is correct for material texture
assets, but IBL resources are environment-derived render resources:

- the diffuse IBL texture is currently generated from an environment-map
  resource descriptor, not a normal `TextureAsset` upload;
- the specular prefilter texture will be generated by renderer-side passes; and
- the IBL sampler is created from descriptor readiness rather than a public
  sampler source asset.

The GLTF scene module-scope `??=` caching is acceptable as an example-only
fixture while the status page proves live allocation and JSON-safe reporting.
It should not become the app's long-term resource lifetime model because it sits
outside `createWebGpuApp()` reuse reporting, app cache summaries, and future
environment resource invalidation.

## Decision

When IBL resources become consumed by StandardMaterial bind groups or shader
sampling, they should move into a renderer-owned app cache adjacent to the
existing texture/sampler caches. The cache keys should be stable environment
resource keys plus preparation role/version tokens, for example:

- `environment-map:<id>:diffuse-ibl@<version>`;
- `environment-map:<id>:specular-prefilter@<version>`;
- `environment-map:<id>:ibl-sampler@<descriptor-version>`.

That keeps GPU resource ownership in `@aperture-engine/webgpu`, avoids ECS-owned
GPU state, and keeps future resource reuse visible in app diagnostics.

## Follow-Up

Add a focused app-cache integration task after the first IBL bind-group layout
slice is selected. It should not block the current descriptor/allocation reports,
but it should land before treating IBL resources as a normal frame-loop input.
