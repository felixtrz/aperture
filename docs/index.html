Render Pipeline State

Detailed comparison: render-pipeline-comparison.html

1. extract

78%

ECS state produces serializable render snapshots with draw, view, light, environment, shadow request, bounds, and diagnostic packets.

  • Missing richer visibility/culling explanations.
  • Missing snapshot diffing and worker transport validation.
  • Missing multi-view render-target coverage beyond current foundations.

2. collect

72%

Asset readiness and source-version dependency summaries exist for meshes, materials, textures, and samplers.

  • Missing one generic change-set scheduler across all prepared assets.
  • Missing complete GLB texture/sampler dependency handoff.
  • Missing automatic unload policy integration.

3. prepare

70%

Prepared mesh/material facades and WebGPU-private backend caches exist for built-in material families, with compact render-world prepared resource summaries and StandardMaterial texture/sampler fidelity diagnostics covered.

  • Missing generic material-family preparation contract.
  • Missing full StandardMaterial PBR texture/resource coverage.
  • Missing IBL, shadow, render-target, and hot-path scratch polish.

4. queue

69%

Material-family queueing, route diagnostics, resource key resolution, Standard transparent phases, app route reporting, and failure diagnostics for frame-resource route shells are implemented; successful frames now route through an internal shell wrapper with omission regression coverage.

  • Missing full frame resource preparation migrated behind the generic route contract.
  • Missing instancing, batching, and multi-material primitive queue rules.
  • Missing mature queue diagnostics for all future material families.

5. sort

58%

Opaque queue sorting is stable and deterministic; alpha-test and transparent phase placement exists.

  • Missing complete transparent ordering policy and tie-break diagnostics.
  • Missing phase-specialized sorting reports.
  • Missing batching and pipeline pressure summaries.

6. submit

55%

WebGPU command submission renders the current proof paths with pipeline, bind group, transform, light, mesh, and material resources.

  • Missing render graph and multi-pass architecture.
  • Missing IBL, shadow, post-processing, and render-target passes.
  • Missing GPU timing and submission reuse diagnostics.
